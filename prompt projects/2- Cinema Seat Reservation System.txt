[PROJECT_OVERVIEW]
The system represents a small cinema hall with a fixed number of seats.

Users should be able to:
- View available seats
- Reserve a seat
- Cancel a reservation
- View seating map
- Exit the program

[SEATING_MODEL]
- The cinema has a grid of seats (e.g., 5 rows × 6 columns).
- Each seat is represented as:
  - "O" = Available
  - "X" = Reserved

Example:
O O O O O O
O O X O O O
O O O O X O

[FEATURES]
Implement the following features:

1. Display Seating Chart
   - Show current seat status in a grid format.

2. Reserve a Seat
   - User selects row and column.
   - If seat is available → reserve it.
   - If already reserved → show error message.

3. Cancel Reservation
   - User selects seat to cancel.
   - Only reserved seats can be canceled.

4. Input Validation
   - Handle invalid row/column inputs.
   - Prevent out-of-range selection.
   - Prevent crashes due to incorrect input types.

5. Seat Availability Check
   - Always check before reserving or canceling.

6. Exit System Safely
   - Gracefully stop the program.
