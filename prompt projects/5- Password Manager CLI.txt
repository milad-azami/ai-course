[CORE FEATURES]

=== FEATURE 1: MASTER PASSWORD ===

On first launch:

* Ask user to create Master Password
* Hash password using a secure algorithm
* Store only hash
* Never store plain text password

On future launches:

* Require Master Password login
* Deny access if password is incorrect
* Allow only 3 login attempts

Requirements:

* Use getpass for hidden password input
* Validate password strength

Password Rules:

* Minimum 12 characters
* Uppercase letter required
* Lowercase letter required
* Number required
* Special character required

=== FEATURE 2: PASSWORD VAULT ===

Each credential record:

{
id,
website,
username,
password,
notes,
created_at,
updated_at
}

Features:

* Add Credential
* Edit Credential
* Delete Credential
* List Credentials
* Search Credentials

=== FEATURE 3: ENCRYPTION ===

All sensitive data must be encrypted.

Encrypt:

* usernames
* passwords
* notes

Requirements:

* Use Fernet encryption from cryptography
* Generate encryption key securely
* Protect stored credentials

Never store plain text passwords.

=== FEATURE 4: SEARCH ===

Allow searching by:

* Website
* Username

Search must be case-insensitive.

=== FEATURE 5: PASSWORD GENERATOR ===

Provide built-in password generator.

User can choose:

* Length
* Include Symbols
* Include Numbers
* Include Uppercase

Generate cryptographically secure passwords.

Use secrets module.

=== FEATURE 6: FILE STORAGE ===

Persist all data locally.

Files:

master.json
vault.json
key.key

Requirements:

* Automatic save
* Automatic load
* Proper error handling
* Corruption detection

=== FEATURE 7: EXPORT ===

Allow exporting vault data.

Export formats:

* JSON

=== FEATURE 8: SECURITY ===

Implement:

* Session timeout

* Secure key handling
* Input validation

Never print sensitive information unnecessarily.
